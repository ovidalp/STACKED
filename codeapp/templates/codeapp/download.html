Dataset Description

Dataset Format

The RGB-D datasets available were captured using the Optitrack Motion capture systems and a Kinect depth camera. A single dataset consist in a gzipped tar file content two folder and three text file. The files rgb.txt and depth.txt list a color images and depth maps grouped into the folders named rgb and depth, respectively.

All data are stored as 640×480 images in PNG format. The color image as 8-bit RGB image and the depth map as 16-bit monochrome images. In the depth map a pixel value of 0 means missing value/no data in the acquisition process, and all other depth values are scaled by 5000 to denote a distance of 1 meter of the camera.

The file groundtruth.txt containing the position of the optical center of the color camera (translation in float format tx ty tz ) and orientation (in quaternion float format qx qy qz qw ) of the camera in a fixed coordinate frame with respect to the world origin defined by the Optitrack motion capture system. Each line in this text file contains a single camera pose with the format : timestamp tx ty tz qx qy qz qw where timestamp represent a number of seconds since the Unix epoch.

Dataset File example

Dataset: ANALISYS

There is also a sequence built for the analysis purpose. It consists of eight poses, each pose is setted for one of the factor-level combination as it shown in the analysis.gif. At each pose,10 captures were performed totaling 80 captures. For each dataset, it is provided the RGB-D images and the Kinect’s camera pose. Both data are synchronized by recording a timestamps at each acquisition.

Filename: ANALYSIS.zip

Archive file format: A .ZIP file containing three text files and eight directories that have been compressed. Each directory have a rgb and depth folders for the corresponding pose.

Packing list:

  • rgb folder: 81 - 8-bit RGB images
  • depth folder: 81 - 16-bit monochrome images
  • Text files: rgb.txt, depth.txt, poses_images.txt

File size: 41.4 MB
